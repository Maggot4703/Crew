# Crew Modularization and Feature Preservation Instructions

## Principles
- **Preserve All Functionality:** Never remove or degrade any existing feature or UI element unless explicitly requested by the user.
- **Use Existing Code:** Always prefer reusing and refactoring existing code over rewriting or introducing new logic, unless a bug or technical debt is being addressed.
- **Incremental Modularization:** Modularize code step-by-step, verifying after each change. Avoid large, sweeping refactors in a single step.
- **Frequent Debugging:** After each modularization or refactor step, run tests or manual checks to ensure no regressions or breakage.
- **Checklist-Driven Planning:** For any modularization or improvement, create a plan for each checklist item and save it to `Crew/plan.md` before implementation.
- **User Corrections are Law:** If the user corrects or undoes an agent change, treat that as a hard rule for future actions.
- **Logging and Progress:** Log all significant actions and progress to `progress.md` and ensure logs are clear and actionable.
- **Respect Project Conventions:** Follow project-specific patterns for CLI/GUI separation, utility extraction, and logging as seen in the current codebase.

## Example Prompts
- "Refactor Crew.py to extract image utilities, but do not remove any features."
- "Plan modularization steps for the checklist and save to plan.md."
- "After each refactor, verify the UI and CLI still work."

## Related Customizations to Consider Next
- File-specific instructions for gui.py or cli.py
- Custom agent for checklist-driven modularization
- Hooks for auto-logging or verification after file edits
